Git Команды

git fetch

git fetch это-Команда которая загружает коммиты, файлы и ссылки из удаленного репозитория в ваш локальный репозиторий. Извлеките данные с помощью команды fetch, если хотите увидеть, над чем работают остальные участники.

git merge

Команда git merge используется для слияния двух веток. Команда git fetch загружает коммиты, файлы и ссылки из удаленного репозитория в ваш локальный репозиторий. Извлеките данные с помощью команды fetch, если хотите увидеть, над чем работают остальные участники.

git rebase

Команда Git Rebase — это мощный инструмент в Git, который позволяет изменять историю коммитов путем перемещения вашей текущей ветки на другой коммит. В отличие от операции слияния (merge) Rebase переписывает историю коммитов..

Разница между ними - это что Merge объединяет истории двух веток, сохраняя их коммиты, в то время как rebase переносит коммиты на новую базу, изменяя историю.

Git Workflow

Git Workflow — альтернативная модель ветвления Git, в которой используются функциональные ветки и несколько основных веток. Эта модель была впервые опубликована и популяризована Винсентом Дриссеном на сайте nvie. По сравнению с моделью магистральной разработки, в Git-flow используется больше веток, каждая из которых существует дольше, а коммиты обычно крупнее.

В Git Workflow создается 5 веток

  • Main - ветка на которой ваш проект
  • Develop - ветка разработки
  • Feature - ветка для разработки новых функций, которые далее сливаются с develop
  • Release - ветка, которая создается, при завершении проектаи и с этого момента новые функции добавить больше нельзя
